Crates

Eight kernel crates behind one facade — each mapping to an OpenCASCADE toolkit, each depending only on the layers beneath it — plus a viewer, 2D sketches, a parametric document layer, and truck interop.

CrateOCCT analogueWhat's insideState
openrcad-foundation TKMath, gp, Precision, Bnd Pnt / Vec / Dir / Ax1/Ax2/Ax3, Trsf, bounding boxes, tolerances, interval arithmetic, exact predicates. tested
openrcad-geom2d TKG2d 2D lines, conics (circle, ellipse, parabola, hyperbola), and NURBS curves via de Boor. tested
openrcad-geom TKG3d, TKGeomBase 3D curves + surfaces: plane, cylinder, cone, sphere, torus, B-spline/NURBS, Bézier/Gregory, offset, ruled. The GeomCurve / GeomSurface owned enums. tested
openrcad-topo TKBRep Arena-based B-Rep: Vertex, Edge, Wire, Face, Shell, Solid on generational slotmap arenas. Per-entity tolerance; validate / is_watertight / manifold_report checks. tested
openrcad-sketch (2D sketch layer) 2D sketch entities — rectangles, circles, lines — resolved into closed Profiles on a SketchPlane. Stores intent for the parametric layer. tested
openrcad-primitives TKPrim make_box, make_cylinder, make_cone, make_sphere, make_wedge — built from the geometry up through Euler operators. tested
openrcad-algo TKBool, TKGeomAlgo, TKFillet SAH BVH, curve/curve/surface intersection (closed-form line/circle fast paths), Euler operators, booleans (fuse/cut/common) with BVH-pruned classification, sewing, and fillet/chamfer/shell_solid. intersection · BVH · booleans tested · box/cylinder blends
openrcad-mesh TKMesh Adaptive face tessellation (chordal + angular) to a TriangleMesh; parallel across faces via rayon. tested
openrcad-exchange TKSTEP, TKSTL STEP AP242 reader/writer (incl. TOROIDAL_SURFACE); ASCII + binary STL writers. tested
openrcad-document (parametric history) Document of sketches + features with insertion-order recompute, and the .zcad serde document format (with cached meshes). The Fusion/FreeCAD-style modeling spine. tested
openrcad-render TKV3d, TKOpenGl Interactive wgpu/winit viewer: orbit/pan/zoom camera, 4× MSAA, topological-edge wireframe overlay, and CPU ray-cast click-to-select with a shader highlight. Not re-exported by the facade — GPU deps stay out of the kernel. viewer milestone
openrcad-truck-compat (interop) Bidirectional truck_topology ↔ openrcad-topo conversion contexts that preserve vertex/edge/face sharing. A separate crate so truck never enters the core graph. conversion
openrcad (facade) Re-exports the kernel as submodules: foundation, geom2d, geom, topo, sketch, primitives, algo, document, mesh, exchange, plus a prelude. (The render and truck-compat crates are intentionally not re-exported.) tested

Picking a dependency

Most applications depend on the openrcad facade and reach in through its submodules. If you want a thinner build — say, only math and topology for a server-side validator — depend on the individual crates directly; the layering guarantees you pull in only what you use.

Every public type is Clone + Serialize + Deserialize. A Solid (and any sub-entity) round-trips through serde_json, bincode, or any serde format — the basis for storing a whole model as one document.
