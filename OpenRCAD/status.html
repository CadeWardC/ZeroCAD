Blending & offset — what works today

Phase 5 geometry is in place, and the blend builders work end-to-end on the two primitive families the detectors recognize. Each returns a watertight solid (sewn) or a typed error.

OperationBoxCylinderConstruction
fillet yes yes Box: cylindrical edge faces + spherical octant corners. Cylinder: torus rolling-ball faces on both rims.
chamfer yes yes Box: ruled bevel faces + triangular corners. Cylinder: 45° conical frustum on both rims.
shell_solid yes yes Offset-surface inner cavity + planar rim faces around the removed open faces.

Scope limit, stated plainly. The blend builders detect a single box or cylinder primitive — at any position/orientation (the frame is recovered from geometry, verified watertight on rotated inputs) — and construct the result directly. For any other solid, including a boolean result, they return BlendError::UnsupportedShape (they never silently hand back the input). The general rolling-ball sweep on arbitrary edges, N-valent Gregory corner blends, face overflow, and concave-offset self-intersection resolution are not yet implemented.

Booleans — robustness frontier (measured)

Booleans run over the BVH-accelerated intersection engine and finish watertight on a wide range of configurations. The split between solid and broken is sharp and tracked by crates/openrcad-algo/tests/robustness.rs against the is_watertight + validate oracles.

Watertight + valid todayOpen frontier (partial imprint)
through-cuts; face-flush unionscorner-overlap unions
enclosed voids; corner-overlap intersectionsblind pockets (tool stops inside)
all rotated placementspartial rotated cuts; through side-drill (non-contiguous loop)

Root cause of the frontier: the split pass does not yet imprint an intersection curve that only partially crosses a face. The failing cases are encoded as #[ignore]d goal-tests (run with cargo test --ignored) so progress is measurable.

New since the last cut

  • ~10× faster booleans — a closed-form curve_curve fast path for line/circle pairs (lines have infinite bounds, so the old subdivision bisected a clamped ±100 domain) dropped box∩box from ~13.8 ms to ~1.3 ms; the classification scan is now BVH-pruned.
  • Watertight / manifold validation — Solid::is_watertight, manifold_report, is_manifold, and health_report, with a regression suite that asserts closure on every verified-good boolean and blend (and pins the failure frontier).
  • Interactive viewer (openrcad-render) — orbit / pan / zoom, 4× MSAA, a topological-edge wireframe overlay, and click-to-select with a shader face highlight.
  • Orientation-general blends — fillet / chamfer / shell verified watertight on rotated boxes and cylinders, not just axis-aligned.
  • Parametric layer — openrcad-sketch (2D profiles) and openrcad-document (feature recompute + the .zcad document format).
  • ToroidalSurface — a first-class analytic surface wired through the geometry enum, the BVH, intersection uv_of, tessellation, and STEP read/write (TOROIDAL_SURFACE).
  • Clippy-clean workspace — every lint addressed; numeric kernels keep index loops by explicit, justified allow.
